PROJECT OVERVIEW
A personalised English + Math learning web app. Multi-user, Google OAuth login via Supabase. Hosted on GitHub Pages (static frontend, no backend server). All data persisted in Supabase.

Live URL pattern: yourusername.github.io/english-app
Auth: Google OAuth via Supabase (no manual accounts)
Database: Supabase (PostgreSQL + Auth + RLS)
AI: Three models switchable globally — Claude Sonnet 4, Gemini 2.5 Flash, Groq (Llama 3.3 70B)

SUPABASE SCHEMA
sql
-- auth.users is auto-managed by Supabase

CREATE TABLE profiles (
  id uuid REFERENCES auth.users PRIMARY KEY,
  name text,
  avatar_url text,
  created_at timestamptz DEFAULT now()
);

CREATE TABLE progress (
  id bigserial PRIMARY KEY,
  user_id uuid REFERENCES auth.users,
  module text,         -- 'phrasal' | 'preposition' | 'idiom' | 'grammar' | 'modal' | 'calc'
  item_key text,       -- e.g. 'Catch', 'Set 1', 'Rule_01_Each_vs_Every'
  question_id text,
  chosen text,
  correct boolean,
  source text,         -- 'static' | 'ai'
  created_at timestamptz DEFAULT now()
);

CREATE TABLE ai_questions_cache (
  id bigserial PRIMARY KEY,
  user_id uuid REFERENCES auth.users,
  module text,
  item_key text,
  model text,          -- 'claude' | 'gemini' | 'grok'
  questions jsonb,
  created_at timestamptz DEFAULT now(),
  UNIQUE(user_id, module, item_key, model)
);

Row Level Security (RLS)
Every table has RLS enabled. Policies allow users to only SELECT/INSERT/UPDATE their own rows (user_id = auth.uid()).

KEY BEHAVIOURS & RULES
All AI questions auto-save to ai_questions_cache on generation; load from cache on revisit

All answered questions (static + AI) save to progress table immediately on answer

Streak increments once per day when ≥1 question is answered; computed from streaks table

Progress bars = distinct correct question_ids / total questions in that module

"Continue" on homepage takes user to last item_key they visited (stored in localStorage as fallback, Supabase as source of truth)

Test modal auto-submits when timer hits 0; per-question time tracked from question render to answer click

Global AI model selector in nav — persisted in localStorage, cycles through models

Difficulty for all AI prompts: SSC-level, tough, avoid trivial questions, include plausible distractors with Hindi hints

Grammar module uses displayKey() to strip Rule_XX_ prefix from rule names in UI

PRACTICE TEST MODULE (mod_practice_test.js)
Features
Full-screen overlay test interface

Loading screen with step-by-step progress (4 steps, % complete)

AI generation error handling with rate limit detection (429)

Question palette with dot grid (numbered, color-coded status)

Per-question timers + global countdown

Mark for review functionality

Results screen with score breakdown + per-question review

Automatic test session saving to Supabase

Marking Scheme
Correct answer: +2 marks

Wrong answer: −0.5 marks (negative marking)

Skipped: 0 marks

Error Handling
Rate limit (429) → shows model-busy error and recommends switching model or retrying

Generation failure → shows error with retry button

Cache check before generation
