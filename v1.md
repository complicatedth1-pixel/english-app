## PROJECT OVERVIEW
A personalised English + Math learning web app. Multi-user, Google OAuth login via Supabase. Hosted on GitHub Pages (static frontend, no backend server). All data persisted in Supabase.

**Live URL pattern:** `yourusername.github.io/english-app`
**Auth:** Google OAuth via Supabase (no manual accounts)
**Database:** Supabase (PostgreSQL + Auth + RLS)
**AI:** Three models switchable globally — Claude Sonnet 4, Gemini 2.5 Flash, Groq (Llama 3.3 70B)

## DATA SHAPES

### Phrasal Verbs (`DB_PV`)
```js
{
  'BaseVerb': {
    intro: 'string',
    verbs: [{ pv, def_en, def_hi, example, note }],
    questions: [{ id, sentence, correct, options:[{verb,hint}], feedback, pdfSource }]
  }
}
```

### Prepositions (`DB_FP`)
```js
{
  'GROUP_NAME': {
    intro: 'string',
    words: [{ word, preposition, def_en, def_hi, example, note }],
    questions: [{ id, sentence, correct, options:[{verb,hint}], feedback, pdfSource }]
  }
}
```

### Idioms (`DB_IDIOMS`) — NO hardcoded questions, AI generates them
```js
{
  'Set 1 (1–20)': {
    setNumber: 1, range: '1–20',
    intro: 'string',
    idioms: [{ id, phrase, meaning_en, meaning_hi, example, note }]
  }
}
```

### Grammar — 12 rules, two-column compare layout
- Wrong sentence vs Correct sentence
- Rule explanation (EN + HI)
- Error spotting question type (click segments A/B/C)

## MODULE STRUCTURE (all modules follow this pattern)
Each module has 3 tabs:
1. **Learn** — content cards (definitions, examples, Hindi meanings)
2. **Practice** — static hardcoded questions + "AI Practice" button (generates 5 AI questions for that specific key/set/rule)
3. **Review** — shows all attempted questions with correct/incorrect badges

## HOMEPAGE (`index.html`)

### Sections (top to bottom)
1. **Sticky nav** — brand logo, user avatar + name (from Google), streak badge, AI model selector pill, logout
2. **Hero section**
   - Greeting: "Good morning, [name]"
   - Streak counter (🔥 N days)
   - Today's goal ring (e.g. do 5 questions today)
3. **Progress section** — 6 cards in 2×3 grid, each with:
   - Module icon + name
   - Progress bar (fills left→right, color matches module accent)
   - `X / Y questions done`
   - "Continue" button → navigates to that module, last item visited
4. **Module nav section** — same 6 modules as large click cards
5. **Test CTA** — prominent button "Take a Test" → opens Test Modal

### Test Modal
Dropdowns in order:
1. **Topic** (single select): English Grammar / Phrasal Verbs / Idioms / Prepositions / Modals
2. **Subtopic** (multi-select, dynamic based on topic):
   - Grammar → list of 12 rules
   - Phrasal Verbs → base verbs (Break, Catch, Give…)
   - Idioms → sets (Set 1, Set 2…)
   - Prepositions → word groups (FROM_Prevention, TO_Attachment…)
   - Modals → modal categories
3. **No. of questions**: 5 / 10 / 15 / 20
4. **Time allocated**: 5 / 10 / 15 / 20 / 30 min
5. **AI Model**: pill toggle — Claude / Gemini / Groq

**Start Test** → full-screen timed test view:
- Countdown timer bar at top
- One question at a time (or all scrollable — TBD)
- Per-question time tracking (time_taken logged per question)
- On time expiry → auto-submit → show results screen
- Results saved to `test_sessions` table in Supabase

## SUPABASE SCHEMA

```sql
-- auth.users is auto-managed by Supabase

CREATE TABLE profiles (
  id uuid REFERENCES auth.users PRIMARY KEY,
  name text,
  avatar_url text,
  created_at timestamptz DEFAULT now()
);

CREATE TABLE progress (
  id bigserial PRIMARY KEY,
  user_id uuid REFERENCES auth.users,
  module text,         -- 'phrasal' | 'idiom' | 'grammar' | 'prep' | 'modal' | 'calc'
  item_key text,       -- e.g. 'Catch', 'Set 1 (1–20)', 'Rule 3'
  question_id text,
  chosen text,
  correct boolean,
  source text,         -- 'static' | 'ai'
  created_at timestamptz DEFAULT now()
);

CREATE TABLE ai_questions_cache (
  id bigserial PRIMARY KEY,
  user_id uuid REFERENCES auth.users,
  module text,
  item_key text,
  model text,          -- 'claude' | 'gemini' | 'groq'
  questions jsonb,
  created_at timestamptz DEFAULT now(),
  UNIQUE(user_id, module, item_key, model)
);

CREATE TABLE streaks (
  id bigserial PRIMARY KEY,
  user_id uuid REFERENCES auth.users,
  date date,
  questions_done int DEFAULT 0,
  UNIQUE(user_id, date)
);

CREATE TABLE test_sessions (
  id bigserial PRIMARY KEY,
  user_id uuid REFERENCES auth.users,
  module text,
  subtopics text[],
  model text,
  total_questions int,
  time_allocated int,   -- seconds
  time_taken int,       -- seconds (actual time used)
  score int,
  results jsonb,        -- [{qid, chosen, correct, time_taken_seconds}]
  created_at timestamptz DEFAULT now()
);
```

### Row Level Security (RLS)
Every table has RLS enabled. Policy: users can only SELECT/INSERT/UPDATE their own rows (`user_id = auth.uid()`).

## KEY BEHAVIOURS & RULES
1. All AI questions auto-save to `ai_questions_cache` on generation; load from cache on revisit
2. All answered questions (static + AI) save to `progress` table immediately on answer
3. Streak increments once per day when ≥1 question is answered; computed from `streaks` table
4. Progress bars = `distinct correct question_ids / total questions in that module`
5. "Continue" on homepage takes user to last `item_key` they visited (stored in localStorage as fallback, Supabase as source of truth)
6. Test modal auto-submits when timer hits 0; per-question time tracked from question render to answer click
7. Global AI model selector in nav — persisted in localStorage
8. Difficulty for all AI prompts: SSC-level, tough, avoid trivial questions, include plausible distractors with Hindi hints
